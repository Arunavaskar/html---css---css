Choosing Images for your site

A picture can say a thousand words and great images help
make the difference between a average looking site and a more engaging one!

Storing Images on your site

I you are building a site from scratch,
it is best to keep all the images that the site uses in a folder as a backup!

As a website grows, keeping all the images in a separate folder helps
understanding how the website is Organizedd.

On a big site you might want to create sub folders for different topic based images,
Like, images for logos and buttons might sit in a folder like interface,
product photograph might sit in a page called products, and images related to
news might sit in a folder called news.

Adding images

"< img >"

To add an image to the website this element is used!
< img > this element hasa no closing tag, which means that this is
an empty tag. It must carry the following attributes...
src, alt and title.

src

This simply tells the browser where to find the picture!
source like a web adress for the imaage location or local
directories location for the image.

alt

alternate text description for the image for the user
if the image is not displayed properly.

title

You can use the title attribute with the
< img > tag to give a more detailed description about the image.
Most browsers will display the title attribute contents when the
user hovers the mouse button over the image.

Height and Width of images

You will often see < img > using two
other attributes as well!

Height

This specifies the Height of the image in pixels.

Width

This specifies the width of the image in pixels.


Images take longer to load compared to HTML code which takes up the rest of the page.
So it is better to specify the size of the image so that the browser can render the page
leaving the right ammount of space for the image to load.

This feature has been removed from HTML and users should

left: align
